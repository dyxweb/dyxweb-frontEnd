## http 报文
>  http 请求报文和响应报文是有一定区别的。每两个部分之间用空格隔开，最后一个部分后面应该接一个换行。

### 起始行
- 请求报文  GET /home HTTP/1.1    方法 + 路径 + http版本
- 响应报文  HTTP/1.1 200 OK  也叫做状态行。由http版本、状态码和原因

### 头部
> 无论是请求头还是响应头，其中的字段都是相当多的，牵扯到http很多特性，字段名不区分大小写，字段名不允许出现空格，不可以出现下划线_，字段名后面必须紧接着:

### 空行
> 用来区分开头部和实体，如果说在头部中间故意加一个空行那么空行后的内容全部被视为实体。

### 实体
> 具体的数据，也就是body部分。请求报文对应请求体, 响应报文对应响应体。
